Необходимо реализовать API для TODO List:
Каждая задача должна иметь следующие свойства:
status (todo, done)
priority (1...5)
title - заголовок задачи
description - описание задачи
expiration_time - срок выполнения. Не обязательное поле.
notify_before - время в минутах. Обязательное, если есть expiration_time
attachment - прикрепленный файл. Не обязательное поле.
Любая задача может иметь подзадачи, уровень вложенности подзадач должен быть неограничен.

У как у клиента API у пользователя должна быть возможность:
- получить список своих задач в соответствии с фильтром
- создать свою задачу
- редактировать свою задачу
- удалить свою задачу
- отметить свою задачу как выпяленную

Фильтр задач должен иметь следующие параметры:
- статус (todo, done)
- приоритет - от и до
- заголовок (должен быть полнотекстовый поиск)
- сортировка по времени создания или времени выполнения или по приоритету

У пользователя не должно быть возможности:
- удалить уже выполненную задачу
- отметить как выполненную задачу у которой есть не выполенные подзадачи

Если есть expiration_time, то пользователю будет приходить имейл за notify_before минут до expiration_time

Формат сдачи задания:
Ссылка на гит репозиторий с выполненным заданием.
Будет плюсом использование Docker

Если будут возникать вопросы по заданию - можно обратиться к HR.
